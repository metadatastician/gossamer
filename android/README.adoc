= Android
:toc: macro

Android platform target for gossamer: a native webview shell *and* native hosts for the non-UI Android components (foreground `Service`, `BroadcastReceiver`, `AppWidgetProvider`), consuming the Idris2 ABI and Zig FFI layers.

== Where the code lives

[cols="1,3"]
|===
| Native (Zig) | `src/interface/ffi/src/`
| — JNI binding | `jni.zig` (real `JNINativeInterface` vtable, pure Zig)
| — WebView backend | `webview_android.zig`
| — component dispatch core | `android_components.zig`
| — component hosts | `android_service.zig`, `android_receiver.zig`, `android_widget.zig`
| Formal ABI (Idris2) | `src/interface/abi/AndroidComponents.idr`
| JVM shims (GENERATED) | `android/gossamer-generated/` (do not edit; `just android-gen`)
| Shim generator | `tools/android-codegen/`
| Design + readiness | `docs/architecture/android-components.adoc`
|===

NOTE: There is no hand-authored Kotlin/Java in this repository. The JVM shims are generated by gossamer's own tooling into `android/gossamer-generated/`, in line with the estate language policy.

== Building

[source,sh]
----
just android-gen      # regenerate the JVM shims + manifest skeleton
just android-build    # cross-compile libgossamer.so for each ABI (needs NDK)
----

See `docs/architecture/android-components.adoc` for the JNI calling convention, the handler-binding model, and the "ready-to-depend-on" checklist for downstream apps (e.g. neurophone#83).
