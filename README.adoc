= Gossamer

image:https://img.shields.io/badge/License-PMPL--1.0-blue.svg[License: PMPL-1.0,link="https://github.com/hyperpolymath/palimpsest-license"]
image:https://img.shields.io/badge/Language-Ephapax-indigo.svg[Ephapax]
image:https://img.shields.io/badge/ABI-Idris2-purple.svg[Idris2 ABI]
image:https://img.shields.io/badge/FFI-Zig-orange.svg[Zig FFI]

_A linearly-typed webview shell with provable resource safety._

**Gossamer** is a desktop application framework that wraps web frontends
(HTML/CSS/JS) in native webview windows, like Tauri or Electron — but with
three guarantees no other framework provides:

1. **Typed IPC** — the frontend-backend protocol is a compile-time type. Mismatches are compile errors.
2. **Linear resource ownership** — webview handles, IPC channels, and plugin resources provably can't leak, double-free, or use-after-free.
3. **Compile-time capabilities** — permissions are linear tokens enforced by the type system, not a runtime config file.

Built in https://github.com/hyperpolymath/ephapax[Ephapax] (a dyadic language with affine + linear types), using the hyperpolymath
https://github.com/hyperpolymath/rsr-template-repo[Idris2 ABI / Zig FFI standard].

== Why Gossamer?

[cols="1,1,1,1,1"]
|===
| | Electron | Tauri | Wails | **Gossamer**

| Resource lifecycle | GC | Arc (refcount) | GC | **Proved (linear types)**
| IPC type safety | None | Codegen | Reflection | **Compile-time**
| Permissions | Opt-in sandbox | Runtime JSON | None | **Linear cap tokens**
| GC required? | Yes | No (Arc ≈ mini-GC) | Yes | **Never**
| Binary size | 150-300MB | 3-8MB | 5-10MB | **1-3MB**
|===

== ABI/FFI Standard

* **ABI (Idris2):** `src/interface/abi/*.idr` — formal specification with dependent type proofs
* **FFI (Zig):** `src/interface/ffi/src/*.zig` — platform-specific C ABI implementation
* **Generated:** `src/interface/generated/*.h` — auto-generated C headers

== Key Files

[cols="1,3"]
|===
| File | Purpose

| `src/interface/abi/Types.idr` | Core types: WebviewHandle, Channel, Cap, WindowConfig, Result codes
| `src/interface/abi/Foreign.idr` | FFI declarations: all C functions with safe wrappers
| `src/interface/abi/Layout.idr` | Memory layout proofs and C ABI compliance
| `src/interface/ffi/src/main.zig` | FFI entry point — platform dispatch, IPC bridge
| `src/interface/ffi/src/webview_gtk.zig` | Linux platform: GTK + WebKitGTK bindings
| `src/interface/ffi/build.zig` | Zig build configuration with platform library linkage
| `examples/hello/main.eph` | Minimal example: window + HTML
| `examples/ipc/main.eph` | Typed IPC with multiple command handlers
| `examples/capabilities/main.eph` | Capability-gated filesystem access
| `docs/decisions/ADR-001-gossamer-webview-shell.adoc` | Full design decision with 14-framework comparison
| `docs/whitepapers/gossamer-arxiv-paper.tex` | Academic paper for arXiv submission
|===

== Hello World

[source,ephapax]
----
import Gossamer.Shell

fn main() -> i32 {
    region app:
        let! webview = Shell.create@app({
            title: "Hello Gossamer",
            width: 800, height: 600,
        })
        let! (webview, _) = Shell.loadHTML(webview, "<h1>Hello!</h1>")
        Shell.run(webview)  // Consumes the handle — can't use it after this
        0
}
----

== Roadmap

* **Phase 1 (v0.1):** Linux — WebKitGTK, typed IPC, capability tokens
* **Phase 2 (v0.2):** macOS + Windows — WKWebView, WebView2, system tray, bundler
* **Phase 3 (v0.3):** Mobile — iOS (WKWebView) + Android (WebView)
* **Phase 4 (v1.0):** Plugin system, auto-updater, full documentation
