= Gossamer Webview Shell — Roadmap

Gossamer is a linearly-typed webview shell built in Ephapax with Idris2 ABI
proofs and Zig FFI. Three novel contributions: linearly-owned webview handles,
typed IPC protocol, and linear capability tokens.

== Current Status

**v0.3.0 released** — Cross-platform desktop support. Linux (WebKitGTK) fully
operational. macOS (WKWebView/Cocoa) implemented with full IPC. Windows
(WebView2/COM) implemented with async COM callback chain and IPC handler.
Platform detection query API for runtime platform introspection.

* [x] 20 exported C-compatible gossamer_* symbols
* [x] Webview lifecycle: create, loadHTML, navigate, eval, setTitle, resize, run, destroy
* [x] Typed IPC: channel open/bind/close with JS bridge injection
* [x] Capability token system: grant/check/revoke with revocation set
* [x] Ephapax modules: Shell.eph, Bridge.eph, Capabilities.eph

== Non-Goals (Explicit)

* Garbage collection — regions + linear types only
* Runtime type information — all types erased at compile time
* Node.js/npm dependency — Deno or nothing
* Electron compatibility layer — clean break, not a shim
